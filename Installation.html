INSTALLATION

mIRC muss portabel installiert sein, das kann man während der Installation auswählen. Sollte das nicht der Fall sein kann man sich eine Verknüpfung machen und das Kommentar -portable dazu geben (nach Rechtsklick>Eigenschaften), dann startet mIRC portabel.

Unterverzeichnis "d-tool" im mIRC Ordner erstellen und die Dateien aus der rar hineinpacken:
d-tool.mrc und d-tool.dialogs.mrc
logo Ordner
languages.db, foreign.db und foreign-mp3.db
rushmirc.dll und rushmirc.dpr für die nutzung von FTPRush

Dann das Script laden über /load -rs d-tool\d-tool.mrc

Geforderte Versionen von FTPRush: ***Rush 1.0.0568 oder höher, aber kein 2.0. und erst recht kein 3.0...
Möchte man mIRC und RushFTP mit einem Windows neuer als 2008 benutzen (wofür die Unterstützung bereits eingestellt wurde), z.B. Windows Datacenter 2019, dann geht meines Wissens nur eine Version welche die rushmirc.dll noch richtig unterstützt und das ist Version 1.1.30 (bestimmt gehen auch andere aus der Zeit)

Wichtige Schritte, die man machen sollte wenn man das Script neu hat, werden hier erklärt

Zum nutzen des IMDb-Checks über OMDbAPI.com muss man sich einen freien Key auf deren Seite holen, den man dann im Kontextmenü unter Settings>IMDb eintragen kann.

Es gibt noch d-tool-remote-at.mrc und d-tool-remotemenü.mrc, die kann man nutzen wenn man d-tool auf einem Server laufen hat und zuhause ebenfalls mIRC startet, dann kann man sich über das mIRC Zuhause direkt Infos von der Remote Box holen ohne per RDP/VNC drauf zu connecten, wie zB. dtinfo, rushstats oder die aktuelle CPU Auslastung.
Dazu muss die d-tool-remote-at.mrc auf der Remote Box zusätzlich geladen werden, die d-tool-remotemenü.mrc braucht man nur, wenn man Zuhause kein d-tool geladen hat. Ansonsten sind die Befehle schon im d-tool Rechtsklickmenü enthalten unter "Remote"
