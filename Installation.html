INSTALLATION

mIRC muss portabel installiert sein, das kann man während der Installation auswählen. Ansonsten macht euch eine Verknüpfung und gebt das Kommentar -portable dazu (nach Rechtsklick>Eigenschaften), dann startet mIRC portabel.

Unterverzeichnis "d-tool" im mIRC Ordner erstellen und die Dateien aus der rar hineinpacken:
d-tool.mrc und d-tool.dialogs.mrc
logo Ordner
languages.db, foreign.db und foreign-mp3.db
rushmirc.dll und rushmirc.dpr für RushFTP
JSONFormIRC.mrc, JSONFormIRC.js und slv-cbapi_moddel.mrc für IMDb Abfragen

Dann das Script laden über /load -rs d-tool\d-tool.mrc

Geforderte Versionen von Rush: ***Rush 1.0.0568 oder höher. Kein Rush 2.0.

Wichtige Schritte, die man machen sollte wenn man das Script neu hat, werden hier erklärt

Zum nutzen des IMDb-Checks über OMDbAPI muss man sich einen freien Key auf deren Seite holen, den man dann im Rechtsklickmenü unter Settings>IMDb eintragen kann.

Es gibt noch d-tool-remote-at.mrc und d-tool-remotemenü.mrc, die kann man nutzen wenn man d-tool auf einer VM laufen hat und zuhause auch mIRC an hat, dann kann man sich über das mIRC Zuhause direkt Infos von der VM holen ohne per RDP/VNC drauf zu connecten, wie zB. dtinfo,rushstats oder die aktuelle CPU Auslastung.
Dazu muss dann die d-tool-remote-at.mrc auf der VM zusätzlich geladen werden. Die d-tool-remotemenü.mrc braucht man nur, wenn man zuhause kein d-tool geladen hat. Ansonsten sind die Befehle schon im d-tool Rechtsklickmenü enthalten unter "Remote"

Möchte man für mIRC und RushFTP ein Windows neuer als 2008 benutzen (wofür die Unterstützung bereits eingestellt wurde), z.B. Windows Datacenter 2019, dann geht meines Wissens nur eine Rush Version die die rushmirc.dll noch richtig unterstützt und das ist Version 1.1.30 (vielleicht gehen auch andere aus der Zeit)
